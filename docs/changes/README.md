# Change Recording Guide

Every meaningful change should be tracked in this folder.

## 1. Files to maintain

- `CHANGELOG.md`: one-line summary per change batch
- `NNNN_description.md`: detailed notes for that batch

## 2. Suggested entry format

1. Date and owner
2. Scope (files/modules touched)
3. Why this change was made
4. What was changed
5. Verification result (tests/demo)
6. Next actions

## 3. Naming convention

- `0001_initial_scaffold.md`
- `0002_phase1_transactions.md`
- `0003_phase1_merkle.md`
- `0004_phase2_blockchain_pow.md`
- `0005_validation_and_docs.md`
